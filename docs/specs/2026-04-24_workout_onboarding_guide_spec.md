# 운동 시작 전 온보딩/가이드 개선 Spec

작성일: 2026-04-24

## 1. 배경

현재 운동 세션 화면에는 운동 시작 전에 필요한 기본 준비 UI가 이미 존재한다.

- 입력 소스 선택: 웹캠 / 화면 공유 / 휴대폰 카메라
- 채점 자세 선택: FRONT / SIDE 등
- 카메라 및 개인정보 안내
- 플랭크 목표 시간 설정
- 운동 시작 버튼

이 방향성은 좋지만, 사용자가 실제 운동을 시작하기 전에 각 운동별로 어떤 자세가 좋은 자세인지, 어떤 자세를 피해야 하는지 알 수 있는 안내가 부족하다. 또한 운동 시작 버튼을 누른 직후 바로 채점이 시작되기 때문에 사용자가 자세를 잡을 시간이 부족할 수 있다.

따라서 준비 화면에 운동별 시작 가이드를 추가하고, 운동 시작 전 3초 카운트다운을 제공해 사용자가 더 안정적으로 운동을 시작할 수 있도록 한다.

## 2. 목표

운동 세션 시작 전에 사용자가 올바른 촬영 환경과 자세 기준을 이해하도록 돕는다.

핵심 목표는 다음과 같다.

1. 운동별 좋은 자세 / 나쁜 자세 예시를 준비 화면에서 제공한다.
2. 전신이 화면에 들어왔는지 확인하는 준비 체크리스트를 제공한다.
3. 운동 시작 버튼 클릭 후 즉시 채점하지 않고 3초 카운트다운 후 시작한다.
4. 초회 사용자에게 짧은 튜토리얼을 표시할 수 있는 구조를 마련한다.

## 3. 구현 범위

### 3-1. 1차 구현 범위

우선 구현할 범위는 아래 3가지다.

| 기능 | 설명 |
| --- | --- |
| 운동별 가이드 박스 | 준비 화면에 스쿼트 / 푸쉬업 / 플랭크별 좋은 자세와 주의할 자세 안내 추가 |
| 준비 체크리스트 | 전신 화면, 조명, 주변 공간, 채점 자세 확인 안내 추가 |
| 3초 카운트다운 | 운동 시작 버튼 클릭 후 3 → 2 → 1 카운트다운 뒤 타이머와 채점 시작 |

### 3-2. 후속 구현 범위

아래 기능은 1차 구현 이후 별도 작업으로 분리한다.

| 기능 | 설명 |
| --- | --- |
| 초회 사용자 튜토리얼 | 처음 접속한 사용자에게만 짧은 안내 modal 표시 |
| 가이드 이미지 / 일러스트 | 텍스트 카드 중심 구현 후 필요 시 이미지 추가 |
| 루틴 단계별 가이드 | 루틴에서 다음 운동으로 넘어갈 때 해당 운동 가이드 표시 |

## 6. 운동별 가이드 박스

### 6-1. 공통 구조

운동별 가이드 박스는 준비 화면에서 현재 운동 코드에 따라 다른 내용을 보여준다.

권장 UI 구조는 다음과 같다.

```text
[운동 시작 가이드]

┌ 좋은 자세 ┐  ┌ 주의할 자세 ┐
│ checklist │  │ warning    │
└───────────┘  └────────────┘
```

또는 모바일에서는 세로 배치로 전환한다.

```text
[운동 시작 가이드]

좋은 자세
- ...
- ...

주의할 자세
- ...
- ...
```

### 6-2. 스쿼트 가이드

#### 좋은 자세

- 발은 어깨너비 정도로 벌린다.
- 무릎은 발끝 방향과 비슷하게 향하게 한다.
- 엉덩이를 뒤로 빼며 앉는다.
- 가능한 범위에서 허벅지가 바닥과 가까워지도록 내려간다.

#### 주의할 자세

- 무릎이 안쪽으로 모이지 않게 한다.
- 상체가 과도하게 앞으로 숙여지지 않게 한다.
- 뒤꿈치가 들리지 않게 한다.
- 너무 얕게 앉으면 반복 인식이 부정확할 수 있다.

### 6-3. 푸쉬업 가이드

#### 좋은 자세

- 손은 어깨보다 약간 넓게 둔다.
- 머리부터 발끝까지 몸통을 일직선으로 유지한다.
- 팔꿈치를 굽혀 가슴이 바닥에 가까워질 때까지 내려간다.
- 올라올 때 몸 전체가 함께 움직이게 한다.

#### 주의할 자세

- 허리가 처지거나 엉덩이가 과하게 들리지 않게 한다.
- 목만 앞으로 빼지 않는다.
- 팔을 너무 좁거나 넓게 짚지 않는다.
- 내려가는 깊이가 너무 얕으면 반복 인식이 부정확할 수 있다.

### 6-4. 플랭크 가이드

#### 좋은 자세

- 팔꿈치는 어깨 아래에 위치시킨다.
- 머리, 등, 엉덩이, 발뒤꿈치가 일직선이 되게 한다.
- 복부에 힘을 주고 자세를 유지한다.
- 목표 시간 동안 호흡을 유지한다.

#### 주의할 자세

- 허리가 아래로 처지지 않게 한다.
- 엉덩이가 과하게 위로 들리지 않게 한다.
- 어깨가 팔꿈치보다 너무 앞이나 뒤로 밀리지 않게 한다.
- 화면에서 몸 일부가 잘리면 자세 판정이 부정확할 수 있다.

### 6-5. 알 수 없는 운동 코드 처리

현재 지원하지 않는 운동 코드가 들어온 경우 공통 안내를 표시한다.

```text
운동 시작 가이드

좋은 자세
- 전신이 화면에 잘 들어오게 위치를 조정하세요.
- 선택한 채점 자세에 맞게 몸을 카메라 방향으로 맞추세요.
- 천천히 정확한 자세로 움직이세요.

주의할 자세
- 화면 밖으로 몸이 벗어나지 않게 하세요.
- 너무 빠르게 움직이면 인식이 불안정할 수 있습니다.
- 주변에 부딪힐 물건이 없는지 확인하세요.
```

## 7. 준비 체크리스트

### 7-1. 표시 항목

준비 화면에 아래 체크리스트를 표시한다.

```text
운동 시작 전 확인
□ 전신이 화면에 들어왔나요?
□ 주변에 부딪힐 물건이 없나요?
□ 조명이 충분하고 몸이 잘 보이나요?
□ 선택한 채점 자세에 맞게 서 있나요?
```

### 7-2. 동작 방식

1차 구현에서는 체크리스트를 비강제 안내로 둔다.

- 체크하지 않아도 운동 시작 가능
- 사용자의 준비를 돕는 안내 역할
- start button disabled 조건에는 포함하지 않음

이유는 다음과 같다.

- 현재도 AI 준비 여부, 플랭크 목표 시간 등 시작 조건이 존재한다.
- 체크리스트까지 필수로 만들면 시작 UX가 무거워진다.
- 데모 / 테스트 중 불필요한 클릭 부담이 생길 수 있다.

## 8. 3초 카운트다운 설계

### 8-1. 현재 시작 흐름

현재 `session-controller.js`의 `startWorkout()`은 대략 아래 순서로 동작한다.

```text
운동 시작 버튼 클릭
↓
AI 준비 여부 확인
↓
세션 시작 API 호출
↓
state.phase = "WORKING"
↓
setup panel 숨김
↓
타이머 시작
↓
포즈 감지 시작
```

### 8-2. 변경 후 권장 흐름

```text
운동 시작 버튼 클릭
↓
AI 준비 여부 확인
↓
세션 시작 API 호출
↓
SessionBuffer 초기화
↓
setup panel 숨김
↓
카메라 overlay에 3초 카운트다운 표시
↓
3
↓
2
↓
1
↓
시작!
↓
state.phase = "WORKING"
↓
타이머 시작
↓
포즈 감지 시작
```

### 8-3. 중요한 동작 규칙

카운트다운 중에는 다음 로직이 시작되면 안 된다.

- 운동 시간 타이머
- rep counting
- pose scoring
- sessionBuffer의 운동 프레임 기록

즉, 카운트다운은 세션 생성 후, 실제 운동 로직 시작 전 구간에 배치한다.

기존 구현에서 `state.phase = "WORKING"`이 타이머와 채점 시작 전에 설정되어 있으므로, 구현 시 아래 두 가지 중 하나를 선택해야 한다.

### 옵션 A: `state.phase = "WORKING"`을 카운트다운 이후로 이동

장점:

- 카운트다운 중에는 명확히 운동 중 상태가 아니다.
- 타이머 조건 `state.phase === "WORKING"`과 의미가 일치한다.

주의점:

- setup panel 숨김, UI 상태 변경 코드 중 `WORKING` 전환에 의존하는 부분이 있는지 확인해야 한다.

### 옵션 B: `state.phase = "WORKING"`은 유지하되 타이머/포즈 감지만 카운트다운 후 시작

장점:

- 기존 상태 전환 영향을 최소화한다.

단점:

- 카운트다운 중 statusBadge가 "운동 중"으로 보일 수 있어 UX 의미가 어색하다.

권장안은 옵션 A다. 카운트다운은 실제 운동 시작 전 준비 단계이므로, `WORKING` 전환을 카운트다운 완료 후로 미루는 것이 상태 의미와 더 잘 맞는다.

## 9. 카운트다운 UI

카운트다운은 기존 `cameraOverlay`를 재사용한다.

예시 구조:

```html
<div class="start-countdown-overlay" aria-live="polite">
  <span class="start-countdown-number">3</span>
  <p>자세를 준비하세요</p>
</div>
```

표시 문구는 아래 흐름을 권장한다.

| 숫자 | 안내 문구 |
| --- | --- |
| 3 | 자세를 준비하세요 |
| 2 | 전신이 화면에 들어오게 유지하세요 |
| 1 | 곧 시작합니다 |
| 시작 | 운동 시작! |

카운트다운 완료 후에는 `cameraOverlay.hidden = true`로 숨긴다.

## 10. 초회 사용자 튜토리얼

초회 사용자 튜토리얼은 후속 구현으로 분리한다.

### 10-1. 저장 키

브라우저 `localStorage`에 아래 키를 저장한다.

```text
fitplus_workout_onboarding_seen_v1
```

### 10-2. 표시 조건

```js
localStorage.getItem("fitplus_workout_onboarding_seen_v1") !== "true"
```

### 10-3. 튜토리얼 내용

```text
FitPlus 운동 세션 안내

1. 영상 원본은 저장하지 않습니다.
2. 전신이 화면에 들어오게 카메라를 맞춰주세요.
3. 운동별 좋은 자세를 확인한 뒤 시작하세요.
4. 시작 버튼을 누르면 3초 후 채점이 시작됩니다.
```

### 10-4. 버튼

```text
확인하고 시작 준비하기
```

클릭 시:

```js
localStorage.setItem("fitplus_workout_onboarding_seen_v1", "true");
```

### 10-5. 주의점

- localStorage 접근 실패는 운동 시작을 막지 않는다.
- 시크릿 모드 / 저장소 비활성 환경에서 조용히 무시한다.
- modal 접근성, 포커스 트랩 등은 후속 구현에서 고려한다.

## 11. 에러 및 예외 처리

### 11-1. AI 모델 준비 전

기존 동작을 유지한다.

```text
AI 모델이 아직 준비 중입니다. 잠시 후 다시 시도해주세요.
```

이 경우 카운트다운은 시작하지 않는다.

### 11-2. 플랭크 목표 시간 미설정

기존 동작을 유지한다.

```text
목표 시간을 먼저 설정해주세요.
```

이 경우 카운트다운은 시작하지 않는다.

### 11-3. 세션 시작 API 실패

세션 시작 API가 실패하면 카운트다운을 시작하지 않는다.

```text
운동 시작에 실패했습니다: ...
```

UI는 기존처럼 복구한다.

### 11-4. 카운트다운 중 페이지 이탈

1차 구현에서는 별도 취소 버튼을 만들지 않는다.

다만 세션이 이미 생성된 뒤 이탈하면 기존 `beforeunload` / `sendAbortBeacon()` 흐름이 처리한다.

1. `runStartCountdown()` helper 추가

4. 카운트다운 중 start button disabled 유지
5. 카운트다운 완료 후 기존 운동 시작 흐름 실행

## 13. 완료 기준

### 13-1. 1차 완료 기준

- 운동 시작 전 준비 화면에서 운동별 가이드가 보인다.
- 스쿼트 / 푸쉬업 / 플랭크마다 다른 안내 문구가 표시된다.
- 알 수 없는 운동 코드에는 공통 안내가 표시된다.
- 전신 화면 준비 체크리스트가 보인다.
- 운동 시작 버튼 클릭 후 3초 카운트다운이 표시된다.
- 카운트다운이 끝나기 전에는 타이머와 채점이 시작되지 않는다.
- 기존 입력 소스 선택, 채점 자세 선택, 플랭크 목표 시간 설정이 정상 동작한다.
- 기존 테스트가 통과한다.

검증 명령:

```bash
npm test
```

## 14. 리스크와 방어 규칙

- 카운트다운 도중 실제 채점이 시작되지 않도록 `startTimer()`와 `startPoseDetection()` 호출 위치를 반드시 카운트다운 이후로 둔다.
- `state.phase = "WORKING"` 전환 시점 변경이 루틴 / 플랭크 UI에 영향을 주는지 확인한다.
- 체크리스트는 1차 구현에서 시작 조건에 포함하지 않는다.
- localStorage 기반 초회 튜토리얼은 저장소 접근 실패가 운동 시작 실패로 이어지지 않게 한다.
- 모바일 화면에서 가이드 카드가 너무 길어지지 않도록 세로 배치와 간격을 조정한다.
