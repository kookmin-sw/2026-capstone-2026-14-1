# 실시간 점수와 반복 확정 점수 UX 정리 계획

- [실시간 점수와 반복 확정 점수 UX 정리 스펙](../specs/2026-04-22-live-score-vs-rep-score-ux-spec.md)

---

## 1. 목표

세션 화면에서 메인 점수 숫자를 `반복 확정 점수` 중심으로 재정의하고, 실시간 점수는 상태형 피드백으로 축소한다.

이번 계획은 구현 순서만 정리한다. 실제 코드 수정은 포함하지 않는다.

---

## 2. 작업 단계

### 단계 1. 현재 점수 표시 책임 분리

- `updateScoreDisplay()`가 담당하는 역할 분해
- 메인 점수 숫자 표시 책임 분리
- 실시간 상태 표시 책임 분리
- breakdown 표시 책임 분리

### 단계 2. 실시간 상태 모델 정의

- 기존 live numeric score를 내부적으로는 유지할지 결정
- 외부 표시용 상태 레벨 정의
  - `GOOD`
  - `WARN`
  - `CORRECT`
- 상태 label, 색상, threshold 매핑 규칙 확정

### 단계 3. 메인 점수 영역 정책 변경

- rep 진행 중 메인 점수 표시 정책 확정
  - 직전 rep 점수 유지
  - 또는 `--` 유지
- rep 완료 후에만 최종 점수 갱신
- rep 시작 시 메인 점수 리셋 여부 결정

### 단계 4. 보조 UI 문구 정리

- rep 시작 전 안내 문구
- rep 진행 중 실시간 상태 문구
- rep 완료 후 확정 점수 안내 문구

### 단계 5. 테스트 추가

- rep 진행 중 메인 점수 불변 검증
- rep 완료 시 메인 점수 갱신 검증
- 실시간 상태 label 매핑 검증
- 기존 결과 저장 흐름 영향 없음 검증

## 4. 예상 리스크

### 리스크 1. 기존 로직과 UI 상태 결합

- 점수 숫자, breakdown, alert가 한 함수 흐름에 묶여 있다.
- 구현 시 점진적으로 분리하지 않으면 조건문만 늘어날 수 있다.

### 리스크 2. 사용자의 숫자 기대치 변화

- 실시간 숫자가 사라지면 일부 사용자는 덜 직관적으로 느낄 수 있다.
- 대체 정보가 충분히 명확해야 한다.

---

## 5. 권장 구현 순서

1. 테스트 먼저 추가
2. 메인 점수 숫자를 rep 완료 점수 기준으로 고정
3. 실시간 상태 label 추가
4. breakdown과 alert 문구 정리
5. 실제 세션 화면 수동 검증

---

## 6. 완료 조건

- 메인 점수 영역 의미가 하나로 고정된다.
- 실시간 교정 기능은 숫자 대신 상태형 피드백으로 유지된다.
- rep 완료 전후 점수 점프가 UX 혼란으로 보이지 않는다.
- 테스트로 진행 중/완료 후 상태가 분리 검증된다.
